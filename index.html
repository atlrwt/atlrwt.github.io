我的第一篇博客

关于博主

大家好，我是 atlrwt ，90后，回炉再造的硕士一枚，目前在日本留学，主攻自动驾驶图像处理方向，爱读书，爱旅游，爱运动，爱游戏。

主力“生产力”工具：MacBook Pro (15-inch, 2017)

主要使用的编程语言：python，C/C++

项目研发平台：Ubuntu 16.04 LTS

GitHub（建设中…）: atlrwt

新浪微博（“微博观察员”，基本只转发游戏相关消息）：atlrwt

关于站名

>一期一会（いちごいちえ）：日本茶道用语，意思是在茶会时领悟到这次相会无法重来，是一辈子只有一次的相会，故宾主需各尽其诚意。

互联网太大了，因此我的博客被人关注到的概率就变得极其小，能够发现它并进来游览一番便是缘分，为了珍惜这份缘分，我愿全力以赴写好每一篇文章，让看到的人能有所收获，不虚此行。

为什么要开通博客

  1. 为了督促自己进行阶段性总结，锻炼写作。

  2. 为了分享经验，结交朋友。

  3. 顺便学习互联网的相关知识。

  4. 趁着2018年初始，斗志满满，抓紧开写。

博客搭建的方案

博客搭建采用 GitHub Pages + Hexo + Markdown 的方案，原因如下：

  1. 因为开源、免费，推荐和维护者众多，出了问题方便解决。
  2. 觉得博客应该以内容为主，一开始如果在搭建方案的选择和配置上花太多时间比较分析，从而耽误了内容创作就本末倒置了，所以采用了比较稳妥的方案。
  3. 用 Markdown 写作无需过多考虑格式上的问题，文章创作过程几乎只靠键盘就能完成，对于喜爱编程或是经常打字的人来讲很容易写出快感（个人见解，反正博主是这样～）。
  4. 写作的同时还可以巩固 Markdown 和 LaTEX 语法，一举多得，岂不美哉。

搭建博客的过程中参考了 diygreen 的免费个人博客搭建详解， zhangslob 的教你免费搭建个人博客，Hexo&Github和小茗同学的使用hexo+github搭建免费个人博客详细教程，避免了不少坑，对三位表示感谢！

博客的内容

  1. 专注于技术分享，现阶段主要是自动驾驶领域。

  2. 项目要用到 ROS，所以会涉及到相关内容。

  3. 说到图像处理，自然少不了目前大热的神经网络了（然而自己也才入门，路漫漫其修远😇）。

  4. 也会夹杂自己平时生活中的感想心得。

  5. 还会安利自己觉得不错的东西/工具。

以上，暂时就这样了。

写在最后

大学开始一直有写日记的习惯，也会做一些总结，但那个内容毕竟还是偏隐私，这次是第一回公开在网上写东西给其他人看，总感觉有些不好意思。

既然第一步已经迈出去了，强迫症表示一定要坚持下去。

今年计划保持每周至少更新一篇文章，看到的朋友请帮忙监督😊

新人上路，请多指教。

祝大家，新年快乐！
